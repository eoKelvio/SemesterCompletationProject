Abstração

A camada de abstração de banco de dados é uma interface de programação que simplifica a comunicação entre uma aplicação e diferentes bancos de dados, como SQL Server, DB2, MySQL, PostgreSQL, Oracle ou SQLite. Ela unifica a interação com os bancos de dados, permitindo que os desenvolvedores escrevam código independente do banco de dados específico utilizado. Isso facilita o desenvolvimento e a manutenção de aplicações, tornando o acesso e manipulação dos dados mais simples e eficiente.

PostgreSQL

Postgre foi o que usamos para fazer as atividades e o projeto. É uma plataforma importante, pois a 2RP usa como ferramenta de banco de dados e é uma plataforma completa, onde podemos fazer todas as operações necessárias para a modelagem do banco de dados. Através dele nós conseguimos modelar o banco de dados da nossa atividade.

Projetos e Atividades

Primeiro Exercício

Esse exercício foi feito para que aprendessemos o básico de como projetar um bando de dados. Tinhamos que criar como se fosse uma livraria virtual, que receberia pedidos de clientes. Foi uma atividade inicial para aprendermos a abstrair e como utilizar as cardinalidades ex(1,n) que seria 1 para vários.

Exercícios de MER

MER (Modelo Entidade-Relacionamento) é um modelo que representa a estrutura de um banco de dados. Ele descreve entidades, seus atributos e relacionamentos. É usado para projetar e organizar os dados, auxiliando no desenvolvimento de sistemas. Nesta atividade tivemos seis exercícios que deveriam ser criados seus MER para que futuramente fizessemos um trabalho modelando um banco de dados em volta deles.

Modelando BD - MER

Dos exercícios anteriores que fizemos, cinco eram exercícios e um era complemento para o primeiro exercício. Foi sorteado dois squads por exercício, onde ambos iriam fazer sem se ajudar e no fim os resultados seriam comparados e análisados. Minha squad ficou com o exercício complementar que em resumo fez com que tivessemos que fazer o MER do primeiro exercício e depois implementar as modificações. Eu fiquei resposável pela montagem do MER, inicialmente tive dificuldade com as relações entre ele, porém quando fizemos em grupo ficou mais fácil.

Modelando BD - O Banco

Tivemos várias aulas sobre como modelar o banco, usamos a plataforma PostgreSQL e todos os integrantes do grupo fizeram essa parte juntos. Um de nós ficou responsável por digitar e salvar os arquivos, enquanto os outros auxiliavam tanto na pesquisa para a criação das tabelas quanto na população delas. Usamos varios comandos vistos na sala de aula como CREATE TABLE, INSERT, NOT NULL, PRIMARY KEY, FOREIGN KEY, etc. Foi a parte mais complicada e demorada do trabalho e foi a parte mais importante também, já que colocamos tudo que vimos na matéria.

Modelando BD - Tabela de Relações

A tabela de relações foi gerada pelo proprio Postgre, porém ficou muito confuso e embaralhado. Então eu decidi refazer ela seguindo tudo que fizemos no banco de dados e no MER, usando o draw.io que foi o site que usamos para fazer todas as atividades até então. No fim a tabela ficou boa e explicava bem como estava funcionando o banco de dados.
